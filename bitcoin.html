Sentiment Analysis and Bitcoin Tweets

Before criticising my dashboard too much, just remember that I am still learning the tips and tricks of Tableau. In addition, this is still a work in progress and is updated regularly.

Project Inspiration

As Bitcoin and echo-chamber-style social media platforms (TikTok, Twitter, etc) have recently exploded in popularity, I became curious to see if sentiment and the price of bitcoin are correlated. During bitcoin's most recent boom, I would scroll through miles of posts with "buy bitcoin" style posts, and I began to wonder: do these posts actually get people to buy fancy internet money (and, in return, drive the price higher)?

To summarize, here's a quick rundown of some of the overarching research questions I worked to answer on this project:

  • How does overall sentiment affect the price of Bitcoin?
  • How does sentiment change geographically?
  • How can we apply time series analysis to draw conclusions about prices?

Quick Rundown

The Twitter API in conjunction to Tweepy was used to scrape 2000 tweets per day. In addition to just tweet text, the following were also collected: User Location, User Follower Count, User Following, User Verification, Tweet Quote Status, Account Creation Date, and Profile Picture Status. These additional features are ultimately used to remove spam from the inital scrape. Results were added into a new csv file each day, and combined into one master csv before additional analysis was done.

The current Bitcoin price was taken from Coindesk. To calculate sentiments, tweets were tokenized and the AFINN lexicon was applied. Scores were averaged each day, resulting in one, overall sentiment. These scores were then graphed against price, as seen in the dashboard.

As user location can be changed manually, locations were changed via python and the str.contains function (ex. New York manually changed to United States, or London changed to United Kingdom). Of course, this is not a perfect solution, but serves more as a bandaid fix and appeared to be effective. The same can be said with the simple spam filter. Spam was attempted to be filtered using Boolean values and account creation date. For the most part, this was effective, but some clearly spam tweets made it through this simple filter.

Future Considerations

Looking back, the whole dataset would be much more effective if all tweets were kept, including spam, and filtered afterwards. This would keep the door open for alternate projects, such as language processing to determine whether or not a tweet is spam.
